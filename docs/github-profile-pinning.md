# GitHub Profile Pinning Guide

Date: 2026-06-13

GitHub profile pins are managed manually through the GitHub UI. Keep the pinned set focused on the strongest six public repositories so the profile stays clear for employers and technical reviewers.

## Recommended Pin Order

1. [commerce-platform-modernization-showcase](https://github.com/shiny-a2/commerce-platform-modernization-showcase) — architecture, migration planning, data modeling, RBAC, and platform modernization.
2. [a2-woocommerce-performance-lab](https://github.com/shiny-a2/a2-woocommerce-performance-lab) — WooCommerce performance, cache boundaries, REST pressure, queue risk, and diagnostics.
3. [a2-crm-operations-system](https://github.com/shiny-a2/a2-crm-operations-system) — CRM operations, operator workflows, provider boundaries, reporting snapshots, and auditability.
4. [commerce-automation-workflows-showcase](https://github.com/shiny-a2/commerce-automation-workflows-showcase) — automation workflows, n8n-style routing, reporting pipelines, and approval gates.
5. [jeweltimeco-showcase](https://github.com/shiny-a2/jeweltimeco-showcase) — retail operations web app, PWA architecture, catalog flow, and assisted ordering.
6. [a2-watch-marketplace-showcase](https://github.com/shiny-a2/a2-watch-marketplace-showcase) — marketplace state design, custody boundaries, review workflows, and settlement readiness.

## Why These Six

- They cover the broadest employer-facing story: architecture, performance, CRM, automation, web apps, and marketplace systems.
- They avoid overcrowding the profile with every active plugin showcase.
- The active plugin/API/UI showcases remain discoverable from the profile README and `docs/public-showcase-index.md`.

## Manual UI Steps

1. Open `https://github.com/shiny-a2`.
2. Click `Customize your pins`.
3. Unpin legacy or narrow repositories that distract from the current full-stack platform story.
4. Pin the six repositories above in the listed order.
5. Re-open the public profile in a private/incognito window and confirm the first-screen story is clear.

## Maintenance Rule

Only change pins when a new repository becomes stronger than one of the six above. For normal active-project updates, update the relevant showcase README, architecture notes, samples, and the full showcase index instead of changing pins.
